## Key Points

Clerk has released Core 3, a major update to their SDKs with improved customization APIs and agent-friendly features
New customization APIs include redesigned hooks (useSignIn, useSignUp, useCheckout, useWaitlist) that simplify building custom auth UIs
Introduction of a visual theme editor allows developers to customize Clerk's prebuilt components and copy configuration directly
Keyless mode now supports additional frameworks: TanStack Start, Astro, and React Router for easier trial without API keys
Modern React compatibility includes support for concurrent rendering features like transitions, Suspense, and streaming SSR
Performance improvements include ~50KB bundle size reduction, faster satellite domains with satelliteAutoSync option, and proactive token refreshing
Simplified package names and unified <Show> component replace multiple previous components (Protect, SignedIn, SignedOut)
Automatic light/dark theme detection and automatic Vite environment variable detection reduce manual configuration
Core 3 requires Node.js 20.9.0+ and includes an upgrade CLI to help migrate from previous versions
Clerk Elements and @clerk/types have been deprecated in favor of new hooks and integrated type exports
