## Key Points

Ray Data LLM is a library specifically designed for large-scale batch inference for LLMs, prioritizing throughput over per-request latency.
Traditional approaches using vLLM's synchronous LLM engine face limitations with large datasets, including memory constraints and lack of streaming execution and fault tolerance.
Ray Data LLM leverages vLLM's asynchronous engine combined with Ray Data's streaming execution to enable processing of datasets larger than available CPU memory.
The solution addresses pipeline inefficiencies caused by variable decode sequence lengths in LLMs through asynchronous batch processing.
Ray Data LLM achieves up to 2x throughput compared to using vLLM's synchronous LLM engine alone.
Key features include continuous batching at the token level rather than request level, disaggregating tokenization/detokenization from the vLLM engine, and production-grade fault tolerance.
The system provides row-level observability and error handling where individual request failures don't crash the entire pipeline.
Asynchronous execution operates at two levels: batch level within Ray Data and engine level within vLLM, eliminating blocking at both batch and token levels.
Performance gains increase significantly with workloads containing mixed decode patterns, particularly when processing long reasoning traces.
Ray Data LLM is modular and can integrate with existing Ray Data pipelines to build complex, large-scale LLM workflows.
