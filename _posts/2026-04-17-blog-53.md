## Key Points

- Cloudflare is addressing OWASP risks in agentic AI systems, focusing on credential leaks, user impersonation, and privilege escalation
- New scannable tokens with 'cf' prefixes and checksums make leaked credentials instantly recognizable to security tools
- Partnership with GitHub enables automatic token revocation when leaks are detected in public repositories
- Enhanced OAuth visibility allows users to see which third-party applications have access to their accounts and easily revoke permissions
- Resource-scoped RBAC expands to more products including Access Applications, Identity Providers, Policies, and Service Tokens
- Cloudflare One customers get additional protection through DLP profiles across network traffic, email, cloud storage, and AI traffic
- GitGuardian reports AI is causing credential leaks to happen 5 times faster than before, with 28+ million secrets published to GitHub last year
- New API token formats are automatically revokable when detected in public repositories, with email notifications to affected users
- Expanded role permissions now available at both account and zone levels for products like Analytics, Security Center, Radar, and more
- Cloudflare recommends customers revoke unnecessary OAuth apps, review API tokens, and adopt granular permission policies for least-privilege security
