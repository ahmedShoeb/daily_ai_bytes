## Key Points

claude-ground is a rule system that gives Claude Code structural discipline it lacks by default
It addresses common issues like losing track of original plans, silent simplifications, not pushing back when needed, and repeating failed approaches
The system includes core rules for phase management, decision logging, honest pushback, debug discipline, and language-specific best practices
Rules are organized into common rules (always active) and language-specific rules for Go, Swift, TypeScript, Kotlin, Flutter, Rust, Python, .NET, and Spring
Rules use severity levels: MUST (hard rules), SHOULD (best practices), and RECOMMENDED (nice-to-haves)
Installation involves cloning the repository and running install.js to set up global rules and project templates
Projects get CLAUDE.md for project context, DECISIONS.md for logging decisions, and phase tracking files to survive context resets
The tool helps maintain project continuity through structured phase management and prevents Claude from deviating from original plans
