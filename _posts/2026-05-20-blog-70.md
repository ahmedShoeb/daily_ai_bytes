## Key Points

Headroom is a context compression layer for AI agents that reduces token usage by 60-95% while maintaining accuracy
It works by compressing everything AI agents read - tool outputs, logs, RAG chunks, files, and conversation history - before it reaches the LLM
The system offers multiple integration options: as a library (Python/TypeScript), proxy server, agent wrapper, and MCP server
Headroom features 'reversible compression' (CCR) where originals are never deleted and can be retrieved by the LLM on demand
It supports cross-agent memory with shared storage across different AI agents like Claude, Codex, and Gemini with auto-deduplication
The compression pipeline includes ContentRouter for type detection, SmartCrusher for JSON, CodeCompressor for AST-based code compression, and Kompress-base for text
Headroom includes a 'learn' feature that mines failed sessions and writes corrections to agent configuration files
Real-world savings examples show dramatic reductions: 92% reduction for code search (17,765 → 1,408 tokens) and SRE debugging (65,694 → 5,118 tokens)
Accuracy remains high with benchmarks showing 97% score on SQuAD v2 QA and BFCL tools benchmarks while achieving up to 32% compression
The system is local-first, runs entirely on your machine, and integrates with major frameworks including LangChain, Agno, Vercel AI SDK, and OpenAI/Anthropic SDKs
