## Key Points

GitHub is announcing a shift from AI as text-based interactions to AI as programmable execution with the new GitHub Copilot SDK
The GitHub Copilot SDK allows developers to embed agentic workflows directly into their applications, rather than just using AI for prompt-response interactions
Teams can now delegate intent to AI agents that can plan steps, invoke tools, modify files, recover from errors, and adapt under defined constraints
Three key patterns for using the Copilot SDK: replacing brittle scripts with adaptable agentic execution, moving from prompt-based logic to structured system integration, and embedding execution into applications beyond the IDE
The SDK makes the same production-tested planning and execution engine that powers GitHub Copilot CLI available as a programmable capability in software systems
This architectural shift moves AI from being a helper tool to becoming infrastructure that can run wherever software operates
The approach addresses common problems like brittle fixed workflows, context-dependency issues, and limited IDE-bound AI capabilities
Agentic execution becomes available through triggers, allowing applications to run AI workflows based on events like file changes, deployment triggers, or user actions
