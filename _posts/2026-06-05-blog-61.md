## Key Points

- Visual AI is shifting from pixel-native generation (diffusion models) to code-native generation, where AI generates the source code behind visual outputs
- Code-native generation produces editable artifacts (SVG, HTML/CSS, React components, Lottie JSON, Blender scripts) rather than just final pixel outputs
- This shift enables continuous iteration, editing, and feedback loops that pixel-native models can't match
- Code-native systems follow a "Code → Render → Inspect → Revise" loop that improves the underlying artifact with each iteration
- 3D asset creation is a promising frontier for visual code generation because 3D assets require consistent underlying structures and functional constraints
- Visual code generation organizes around different runtime environments (browsers, SVG renderers, Lottie players, Blender, game engines)
- The future will likely be hybrid, with pixel-native models handling realism and code-native systems handling structure and production workflows
- Winning products in this space will own the entire generation-inspection-revision loop rather than just generating outputs
