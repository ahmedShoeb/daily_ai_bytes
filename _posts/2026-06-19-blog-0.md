## Key Points

- OpenAI has added Chrome DevTools Protocol support to Codex, allowing it to access web browsers for code generation and live interaction
- Codex can now profile JavaScript performance, read console output, monitor network traffic, analyze page payloads, and inspect rendered states through CDP
- The AI can reach into and modify a website's DOM, enabling live page customization like theme recoloring, font adjustments, and data extraction
- This browser integration is currently opt-in, available under Settings, but organizations can disable it if needed
- The feature is not available in the European Economic Area, UK, or Switzerland at launch
- Implementation is still early stage - runs slowly, can overload under pressure, and sometimes requires restarts
- OpenAI previously acquired development environment company Ona (formerly Gitpod) to support longer-running coding tasks
- Codex now has over 5 million weekly users and aims to create an AI layer that sits between users and the web
- The vision requires faster models and infrastructure that don't yet exist at scale
