## Key Points

- Exa AI built Canon, a search pipeline orchestrator to handle the growing complexity of their search engine serving billions of requests
- Canon represents search pipelines as Directed Acyclic Graphs (DAGs), enabling automatic parallelism, durable execution, and introspectability
- The DAG approach allows independent nodes to run concurrently, with the runtime handling dependencies, cancellation, and memoization automatically
- A core goal of Canon is providing visibility across full search paths with detailed tracing for debugging billions of queries
- Canon evolved from an imperative codebase with manual error checks to a structured graph system where the runtime handles orchestration
- The architecture enables a clean separation of concerns where nodes focus on business logic while the runtime manages tracing, caching, and parallelization
- Canon addresses challenges of AI agent-written code by making implicit search path rules explicit through typed execution graphs
- The system ensures correctness through totality - every node must handle all possible outcomes, eliminating unhandled edge cases
- This approach is particularly important as Exa serves thousands of different AI agents, each with unique search requirements
- Canon allows traceability for debugging why specific URLs were dropped or why search results behave unexpectedly
