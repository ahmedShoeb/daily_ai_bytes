## Key Points

Google has introduced a new 'subagents' feature in Gemini CLI to enable parallel coding task execution
Subagents allow work to be split across multiple specialized agents within the same environment, each with their own instructions, tools, and context
This addresses the bottleneck where AI coding agents work through tasks one at a time when tasks stack up
The main agent can delegate parts of tasks to subagents, allowing simultaneous execution of different components like frontend updates, testing, and documentation
Subagents run in their own working space, reducing the risk of tasks interfering with each other in complex sessions
Developers can create custom subagents using Markdown files with YAML frontmatter for specific roles like code review, testing, or documentation
Gemini CLI includes built-in subagents: a 'generalist' agent, a CLI-focused agent, and a codebase-focused agent for exploring architecture and debugging
Tasks can be automatically routed to appropriate subagents or explicitly assigned using the @ syntax followed by the agent's name
The /agents command lists configured subagents available in the current session
This approach differs from systems like Claude Code's 'agent teams' which coordinate work across multiple sessions rather than within a single session
