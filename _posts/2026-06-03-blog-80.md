## Key Points

- Nvidia has released a Model Card Generator (MCG) toolkit that automates AI model documentation in under a minute to address growing regulatory requirements like California's AB-2013 and EU AI Act
- The toolkit follows an Ingestion → Extraction → Rendering pipeline that processes model source code from URLs (GitHub, GitLab, HuggingFace) or uploaded files (ZIP, PDF, DOCX, Markdown)
- It uses retrieval-augmented generation (RAG) with Nvidia Inference Microservices (NIM) and GPT-OSS-120B to extract structured JSON data that converts into Model Card++ format
- The system generates a complete model card including overview plus four subcards: Bias, Explainability, Privacy, and Safety & Security
- Designed for flexibility with customizable models, templates, and field-level guidance - can adapt to different regulations and industries without changing core code
- Runs as containerized services with no proprietary cloud lock-in - can be deployed on-premises or in your own cloud with Kubernetes support
- Performance testing shows it generates full model cards in under a minute with 91% completion rate and 76% accuracy across standardized test sets
- When documentation is sparse, the system surfaces 'not found' or 'information not available' rather than guessing, making it useful for identifying documentation gaps
- Oracle has integrated the MCG toolkit into their OCI AI offering, deploying it with Llama-3.3-Nemotron-Super-49B-v1 as the core extraction model
- Early adopters can reach out to Nvidia's Trustworthy AI team, while others can use open source Model Card++ templates available in the Trustworthy AI GitHub repository
