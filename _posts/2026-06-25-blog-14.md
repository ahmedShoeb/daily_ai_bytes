## Key Points

- Claude Code records session logs to disk including "thinking blocks" that are actually encrypted signatures (600 characters long) with no accessible text
- The reasoning is encrypted into signatures that only Anthropic holds the key to - users' machines don't receive the actual reasoning
- The API returns a SUMMARY of reasoning, not the actual reasoning itself
- Getting the full thinking output requires an enterprise agreement
- The "extended-thinking" output (ctrl+o) is a summary of thinking logic, not the actual thinking that drove the model's actions
- Users cannot produce the actual reasoning logic from local files - the reasoning logs on their systems are not accessible
- Users can only log inputs, outputs, and actions through workarounds but still won't get the actual reasoning
- The author warns developers to be aware before promising audit trails to clients
- The documentation language is described as "awfully indirect" and easy to misunderstand without careful reading
