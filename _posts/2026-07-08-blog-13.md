## Key Points

- The article shares insights from a Fireside Chat with Cat Wu and Thariq Shihipar from the Claude Code team at AIE
- Key recommendation: Let AI models like Fable and Opus use their own judgement rather than micromanaging how they should work
- Example given: Instead of specifying when to write tests, just tell the AI to use its own judgement about when testing is appropriate
- Jesse Vincent provided additional advice to avoid burning expensive Fable tokens: have Fable delegate smaller tasks to lower-power models
- Simon Willison created a specific prompt: 'For all coding tasks use your judgement to decide an appropriate lower power model and run that in a subagent'
- Claude automatically saved this instruction as a memory file in the project system
- The approach aims to keep judgment, review, and synthesis tasks with the main model while delegating implementation work to cheaper models
- Simon reports this method is working well - he's getting substantial work done while using fewer expensive Fable tokens
- This strategy helps optimize costs before anticipated AI model price increases
