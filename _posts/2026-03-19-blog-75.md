## Key Points

- OpenAI Codex subagents have reached general availability after several weeks of preview
- The subagent system is similar to Claude Code's implementation with default subagents for 'explorer', 'worker', and 'default'
- The 'worker' subagent appears designed for running large numbers of small tasks in parallel based on CSV examples
- Codex allows users to define custom agents as TOML files stored in ~/.codex/agents/
- Custom agents can have custom instructions and be assigned to use specific models including gpt-5.3-codex-spark for raw speed
- Users can reference custom agents by name, as shown in documentation example for debugging UI issues
- The subagents pattern is now widely supported across multiple coding agent platforms
- Documentation exists for subagents in OpenAI Codex, Claude Code, Gemini CLI (experimental), Mistral Vibe, OpenCode, Visual Studio Code, and Cursor
- Simon Willison has added a chapter on Subagents to his Agentic Engineering Patterns guide
