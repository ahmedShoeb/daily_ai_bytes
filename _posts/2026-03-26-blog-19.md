## Key Points

NVIDIA's new method allows you to transform a general-purpose embedding model into a domain-specific one in under a day using a single GPU with at least 80GB memory
The approach eliminates manual labeling by using LLMs to automatically generate synthetic question-answer pairs from domain documents through a four-stage pipeline
Hard negative mining is crucial - it identifies confusing passages that look relevant but aren't correct answers, forcing the model to learn subtle distinctions in the domain
The method includes multi-hop query generation (1-3 hops) to handle complex questions that require synthesizing information across multiple documents
Users can fine-tune the base Llama-Nemotron-Embed-1B-v2 model, achieving 10%+ improvements in Recall@10 and NDCG@10 metrics
Real-world validation by Atlassian showed Recall@60 jumping from 0.751 to 0.951 (26% improvement) on Jira data using a single A100 GPU
The pipeline includes export to ONNX/TensorRT for production optimization and deployment via NVIDIA NIM containers with OpenAI-compatible APIs
NVIDIA provides a ready-to-use synthetic training dataset generated from their public documentation to help users get started quickly
The entire process requires six commands and can complete in 2-3 hours for a small corpus (~500 documents)
The solution is designed for integration into existing RAG pipelines with minimal code changes, using standard formats like JSON, BEIR, and ONNX
