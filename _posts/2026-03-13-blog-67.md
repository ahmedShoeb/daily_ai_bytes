## Key Points

- An agent consists of two parts: Model + Harness, where the harness is every piece of code, configuration, and execution logic that isn't the model itself
- The harness provides essential capabilities that models cannot do out of the box: durable state storage, tool execution, real-time knowledge access, and environment setup
- Core harness components include filesystems for durable storage, bash/code execution tools for autonomous problem-solving, and sandboxes for safe execution environments
- Harnesses combat 'context rot' through techniques like compaction (summarizing context), tool call offloading, and progressive disclosure of tools via skills
- For long-horizon autonomous execution, harnesses enable planning, self-verification loops, and Ralph Loops that force agents to continue work against completion goals
- There's a co-evolution between model training and harness design, where useful primitives are discovered in harnesses and then used to train better models
- While models will absorb some harness functionality over time, harness engineering remains valuable for building effective agents, similar to how prompt engineering remains relevant
- Current research areas include orchestrating hundreds of parallel agents, agents that analyze their own traces to fix harness-level failures, and dynamic tool assembly
