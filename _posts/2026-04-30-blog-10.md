## Key Points

OpenAI introduces Symphony, an open-source agent orchestrator that transforms issue trackers into control planes for AI coding agents
Symphony automatically assigns agents to every open task and manages their work continuously, reducing human supervision overhead
Some OpenAI teams saw a 500% increase in landed pull requests within the first three weeks of using Symphony
The system emerged from OpenAI's experience building projects with no human-written code, relying entirely on Codex-generated code
Symphony treats Linear (project management tool) as a state machine, where each open issue gets its own agent workspace
Agents can break down complex tasks into subtasks with dependencies, ensuring optimal parallel execution
The orchestrator handles routine development workflow tasks like rebasing, conflict resolution, and CI shepherding
Symphony is essentially just a SPEC.md file with a reference implementation in Elixir, making it easy to adapt with coding agents
The system uses Codex App Server with JSON-RPC API for programmatic interaction instead of CLI sessions
Symphony demonstrates how the bottleneck in AI-assisted development shifts from code writing to work management
