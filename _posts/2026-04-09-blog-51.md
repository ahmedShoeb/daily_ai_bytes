## Key Points

Google is building 'Jitro' as the next generation of its Jules coding agent
Jules V2 moves beyond prompt-and-execute models to high-level goal-setting
The agent autonomously identifies what needs to change in a codebase to improve metrics
This represents a departure from task-level paradigms used by competitors like GitHub Copilot and OpenAI's Codex
Google envisions Jitro as a persistent collaborator with a dedicated workspace rather than a one-shot tool
The agent includes tools for listing goals, tracking insights, and managing tool integrations
Expected launch timing aligns with Google I/O 2026 (May 19, 2026)
Primary beneficiaries would be engineering teams managing large codebases needing performance optimization and compliance improvements
The main risk involves autonomous goal-pursuing agents introducing unpredictable changes
Google sees this as the next direction for AI-assisted development
