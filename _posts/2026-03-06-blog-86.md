## Key Points

The author conducted a follow-up review of OpenAI Codex from daily production use at WorkOS and personal projects, showing dramatic improvements since their 2025 review
Codex success rates for well-scoped maintenance tasks have jumped from 40-60% to 85-90%, transforming the author's daily workflow
At WorkOS, the author uses Codex to batch 4-5 maintenance tasks first thing in the morning, freeing up 30-40% of development time previously spent on SDLC grunt work
Codex excels at extending established patterns in mature codebases, understanding existing architectures and maintaining consistent code styles
Major improvements include dramatically better error handling with clear messages, multi-turn conversations for PR refinement, and a new preview system showing 2-4 implementation approaches
The network connectivity issue from the original review has been largely fixed with granular controls for sandbox network access
The author notes Codex appears to be training on its own usage data, with systematic, automated refinement leading to consistent improvements
Despite improvements, the author remains frustrated with model selection opacity - they can't choose which model handles specific tasks based on complexity
Codex has fundamentally changed the author's workflow, handling maintenance tasks, backlog fixes, and implementation details while freeing up time for architectural problems
The author concludes Codex has crossed the threshold from experiment to indispensable tool and recommends adopting it for serious development work
