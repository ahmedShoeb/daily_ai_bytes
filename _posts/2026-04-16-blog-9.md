## Key Points

- Kiro CLI 2.0 introduces three major updates: headless mode for CI/CD pipelines, native Windows support, and a UX refresh
- Headless mode allows programmatic access to Kiro CLI for automated workflows and CI/CD integration without interactive sessions
- Native Windows support eliminates the need for WSL workarounds, providing direct access to Kiro agents within Windows Terminal
- The UX refresh includes a new subagent experience with task lists for monitoring progress and improved agent management
- Subagents can be used to parallelize work while protecting parent agent context, with designer → implementer → reviewer workflows
- Task lists provide real-time updates on agent progress and accomplishments during complex tasks
- The CLI 2.0 is now the default experience, but users can revert to classic mode with 'kiro-cli --classic' if needed
- Installation can be done via curl command: 'curl -fsSL https://cli.kiro.dev/install | bash'
- The update was driven by user feedback gathered since the initial launch, addressing workflow integration needs
