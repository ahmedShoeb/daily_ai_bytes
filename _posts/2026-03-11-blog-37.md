## Key Points

- Google senior AI product manager Shubham Saboo has open-sourced an 'Always On Memory Agent' on Google Cloud Platform's GitHub under MIT License
- The agent provides persistent memory for AI systems without traditional vector databases - using SQLite storage and scheduled memory consolidation every 30 minutes
- Built with Google's Agent Development Kit (ADK) and Gemini 3.1 Flash-Lite, Google's fastest and most cost-efficient model released March 2026
- The system ingests text, images, audio, video, and PDF files continuously and consolidates memories in the background
- Key design claim: 'No vector database. No embeddings. Just an LLM that reads, thinks, and writes structured memory'
- Gemini 3.1 Flash-Lite makes the always-on approach economically feasible at $0.25 per 1M input tokens and $1.50 per 1M output tokens
- Enterprise concerns focus on governance - compliance risks when agents 'dream' and cross-pollinate memories without deterministic boundaries
- The system uses a multi-agent architecture internally but is specifically an always-on memory agent rather than a shared framework for multiple independent agents
- The release signals a shift toward treating agents as deployable software systems rather than isolated prompts, with memory becoming part of the runtime layer
