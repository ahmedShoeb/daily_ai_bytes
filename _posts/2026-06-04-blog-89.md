## Key Points

Perplexity is introducing Search as Code (SaC) - a new architecture that treats search as programmable primitives rather than monolithic services
Traditional search pipelines are increasingly outdated in the era of AI agents that need to complete complex tasks with thousands of retrieval operations
SaC exposes search building blocks through an Agentic Search SDK that agents can compose via code generation in secure sandboxes
This approach solves three main problems with traditional search: coarse context, failure to leverage domain knowledge, and inefficient control flow with context pollution
The architecture consists of three layers: models as control plane, compute sandboxes for execution, and the Agentic Search SDK for search primitives
In benchmark testing, SaC outperformed competitors across multiple domains including DSQA, BrowseComp, WideSearch, and the new WANDR benchmark
The WANDR benchmark showed particularly dramatic results - SaC outperformed the next-best system by 2.5x
SaC achieved 100% accuracy on a real-world CVE vulnerability analysis task while reducing token usage by 85.1% compared to non-SaC baselines
This architecture represents a shift toward hybrid computing that combines token-space reasoning with deterministic runtime execution
Perplexity is rolling out SaC capabilities starting with Perplexity Computer and Agent API
