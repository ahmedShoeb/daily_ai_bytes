## Key Points

- This article reveals that continual learning for AI agents happens at three distinct layers: the model, the harness, and the context.
- The model layer refers to updating the actual model weights, which is what most people think of when discussing continual learning.
- The harness layer involves the code and infrastructure that drives all instances of an agent, which can be optimized based on performance traces.
- The context layer consists of external configuration elements like instructions, skills, and memory that configure the harness for specific use cases.
- Open examples include Claude Code and OpenClaw, which use similar three-layer architectures with model, harness, and user/agent contexts.
- A key challenge at the model layer is catastrophic forgetting, where models degrade on previous knowledge when updated with new data.
- Harness optimization can use techniques like Meta-Harness, which analyzes traces to suggest improvements to the harness code.
- Context learning can happen at agent, user, or organizational levels and can be updated either offline ("dreaming") or live during execution.
- Traces from LangSmith power all these learning processes by capturing complete execution paths for analysis and improvement.
- All three learning layers can be combined in practice, with Deep Agents serving as a production-ready harness supporting these capabilities.
