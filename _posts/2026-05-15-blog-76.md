## Key Points

- Researchers RL fine-tuned 4B parameter models to behave as native recursive language models (RLMs) that can recursively call themselves
- The approach trains a single policy to handle both parent RLMs (which decompose tasks) and child RLMs (which execute sub-tasks)
- The system uses a Python REPL environment where models write and execute code to inspect data, with built-in functions for spawning child RLMs
- Focused on evidence selection from scientific papers where RLMs extract relevant snippets from multiple documents
- The training setup allows child RLM rollouts to inherit advantages from parent RLMs, enabling single-policy learning
- Results show RL fine-tuned 4B models perform comparably to larger models like Claude Sonnet 4.6 but at much lower cost and latency
- The trained model achieves 7-second wall-clock time per query vs 60+ seconds for Sonnet-based RLMs
- The approach requires an initial SFT (Supervised Fine-Tuning) phase as RLMs are outside the competence edge for most small models
- Training uses rubric-based LLM judges for reward assignment rather than verifiable rewards to avoid reward hacking
- Future directions include task exploration, better reward assignment, and scaling to discover new strategies automatically
