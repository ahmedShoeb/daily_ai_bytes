## Key Points

- This article outlines five multi-agent coordination patterns for AI systems and when to use each one
- Generator-verifier pattern involves one agent generating output and another verifying it against explicit criteria, best for quality-critical tasks like code generation or fact-checking
- Orchestrator-subagent pattern uses a hierarchical structure where a lead agent delegates tasks to specialized subagents, ideal for clear task decomposition like automated code review systems
- Agent teams pattern features persistent worker agents that handle independent, long-running tasks, suitable for parallel work like codebase migrations where each agent retains context
- Message bus pattern uses event-driven communication where agents publish and subscribe to events, fitting for growing ecosystems like security operations automation
- Shared state pattern allows agents to coordinate through a persistent store that all can read/write directly, best for collaborative research where findings build on each other
- The article recommends starting with simplest pattern (usually orchestrator-subagent) and evolving based on specific needs and limitations observed
- Choice depends on factors like task independence, workflow predictability, need for sustained context, and information sharing requirements
- Patterns can be combined in production systems, with common hybrids using orchestrator-subagent for workflows with shared state for collaboration-heavy subtasks
- Each pattern has specific failure modes and requires careful design decisions around termination conditions, routing accuracy, and information flow management
