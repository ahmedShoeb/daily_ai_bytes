## Key Points

- LLM architectures are increasingly focused on long-context efficiency as reasoning models and agent workflows keep more tokens active
- Key architectural innovations include KV sharing/reuse across layers in Gemma 4 models to reduce KV cache memory
- Per-layer embeddings (PLE) in Gemma 4 E2B/E4B models add capacity through layer-specific embedding slices without scaling transformer stack size
- Laguna XS.2 uses layer-wise attention budgeting where different layers have varying numbers of query heads based on their attention type
- ZAYA1-8B introduces Compressed Convolutional Attention (CCA) that performs attention operations directly in compressed latent space
- DeepSeek V4 implements manifold-constrained hyper-connections (mHC) for more expressive residual pathways with minimal overhead
- DeepSeek V4 also uses Compressed Sparse Attention (CSA) and Heavily Compressed Attention (HCA) that compress along the sequence dimension
- Architecture innovations are focused on reducing long-context costs, KV cache size, and memory traffic while maintaining quality
- DeepSeek V4 achieves significant efficiency gains: DeepSeek V4-Pro uses only 27% of single-token inference FLOPs and 10% of KV cache size compared to V3.2
- The transformer block continues to evolve with targeted tweaks rather than complete replacement, adding complexity but reducing runtime costs
