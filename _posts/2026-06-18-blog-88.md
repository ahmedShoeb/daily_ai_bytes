## Key Points

- GitHub has released the GitHub Multilingual Repositories Dataset, a repository-level metadata dataset designed to help discover public GitHub repositories with non-English content
- The dataset covers over 80 million classification rows across more than 40 million repositories, providing language classifications for READMEs, issues, and pull requests
- Researchers found that language distribution differs across content types: Korean is most common in issues but only fifth-most common in READMEs, while Portuguese tops READMEs with over 3 million repositories
- The dataset is released under CC0-1.0 license and follows Microsoft's European Digital Commitments to make multilingual data more accessible to open source AI developers
- The dataset includes classifications from three different language identification tools (fastText, gcld3, and lingua-py) with confidence scores, allowing users to set their own precision thresholds
- Key applications include discovering repositories with specific language content, studying non-English developer communities, building evaluation sets for AI coding tools, and measuring representation of underrepresented languages
- The dataset addresses the challenge that many European languages remain underrepresented in online text used to build AI systems, creating risks of unequal AI tool performance across languages
- GitHub emphasizes that repository text presents unique challenges for language identification due to short texts, badges, templates, code snippets, and mixed-language content
- The dataset is designed as a transparent discovery tool rather than a ground-truth benchmark, allowing users to inspect classifications and confidence scores
- GitHub will showcase the dataset at an Open Innovation Dialogue Hub event in Strasbourg on June 16, co-organized with Microsoft and the Council of Europe
