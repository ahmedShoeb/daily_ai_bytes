## Key Points

Google is announcing new capabilities for Managed Agents in Gemini API to address developer feedback and product needs
Key new features include: background execution for long-running tasks, remote MCP server integration, custom function calling alongside sandbox tools, and network credential refresh
Background execution allows agents to run asynchronously on servers instead of holding HTTP connections open, with APIs returning an ID for polling and reconnection
Remote MCP server integration enables agents to connect directly to private databases and internal APIs without custom proxy middleware
Custom function calling allows mixing custom tools with built-in sandbox tools for local execution alongside server-side operations
Network credential refresh allows updating access tokens and API keys while maintaining sandbox state, installed packages, and cloned repositories
These updates transform managed agents into asynchronous workers that can operate inside real development environments without blocking applications
Developers can get started with the Gemini Interactions API overview and managed agents quickstart to explore these new capabilities
