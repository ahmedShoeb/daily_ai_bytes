## Key Points

- TorchTPU is Google's new system allowing PyTorch to run natively on Tensor Processing Units (TPUs)
- It enables developers to migrate existing PyTorch workloads with minimal code changes - just change initialization to 'tpu'
- TorchTPU follows an 'Eager First' philosophy with three eager modes: Debug Eager, Strict Eager, and Fused Eager
- Fused Eager mode delivers 50% to 100+% performance increase by automatically fusing operations for better TensorCore utilization
- For peak performance, TorchTPU integrates with torch.compile using XLA as the backend compiler
- Supports Distributed Data Parallel (DDP), Fully Sharded Data Parallel v2 (FSDPv2), and PyTorch's DTensor out of the box
- Unlike predecessor PyTorch/XLA, TorchTPU supports divergent executions (MPMD) for more natural PyTorch workflows
- TPUs feature TensorCores for dense matrix math and SparseCores for irregular operations like embeddings
- TPU systems use Inter-Chip Interconnect (ICI) in 2D/3D Torus topology for efficient scaling
- Roadmap includes reducing recompilations for dynamic sequences, comprehensive precompiled kernels library, and hardware-specific optimization guidance
