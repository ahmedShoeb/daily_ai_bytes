## Key Points

Traditional search systems are monolithic stacks with separate components (embeddings, rerankers, query understanding) that operate independently without seeing the whole picture
Agentic search models unbundle this architecture by having a single intelligent model orchestrate the entire search process using available tools
Frontier lab models like GPT-5 work well for 80% of cases but struggle with the last 20% - domain-specific knowledge and user behavior patterns
New agentic search models like SID-1, Glean's Waldo, and startups like Charcoal are being trained specifically for search tasks in specific domains
These specialized models can be smaller, faster, and more efficient than general-purpose models like GPT-5 for search applications
Future search may move from complex query/reranking pipelines to simple retrieval primitives orchestrated by specialized search agents
Just as we now have domain-specific embedding models, we'll likely see a proliferation of agentic search models tuned for different domains like e-commerce, legal, or job search
These models could potentially encompass the entire search process from query understanding to hybrid search, though current latency needs improvement for production use
