## Key Points

- MCP (Model Context Protocol) is being overshadowed by Skills in the AI space, but the author prefers MCP for service integration
- MCP provides API abstraction where LLMs call functions like devonthink.do_x() while servers handle implementation details
- MCP offers advantages: zero-install remote usage, seamless updates, OAuth-based authentication, cross-platform portability, and sandboxing
- Skills work well for pure knowledge and teaching existing tools, but struggle when requiring CLI installation
- Skills have deployment issues: CLI installation problems, secret management nightmares, fragmented ecosystems across different AI tools
- Skills can bloat context windows by loading entire SKILL.md files rather than specific tool signatures
- The author suggests MCP should connect services while Skills should focus on knowledge and workflow standardization
- Services like Google Calendar, Chrome, and Xcode should provide native MCP endpoints instead of CLI-based Skills
- The author has built services (microfn, Kikuyo, MCP Nest) using MCP architecture for better remote access
- MCP Nest tunnels local MCP servers through the cloud to make them accessible from any MCP-capable client
