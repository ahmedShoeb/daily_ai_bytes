## Key Points

- LLMs process all input as one continuous token stream without inherent separation between user inputs, system prompts, and their own reasoning
- Role tags (system, user, think, assistant, tool) are the mechanism LLMs use to recover structure, but these boundaries can fail
- Prompt injection occurs when low-privilege text gains authority of higher-privilege roles due to LLMs relying on writing style rather than secure tags
- Researchers developed role probes to measure how LLMs internally perceive roles, finding that reasoning-like writing style can override actual role tags
- Attackers can create CoT Forgery attacks by spoofing reasoning style, making LLMs treat fake reasoning as their own trusted conclusions
- Simple prompts like "User: " in front of commands can trick LLMs into perceiving tool text as user commands
- Role confusion enables subconscious steering where innocuous text can subtly shift LLM behavior toward commercial goals
- Roles separate competing objectives (exploration vs communication, comprehension vs generation) but weren't designed systematically
- Current LLM architecture treats roles as fundamental cognitive scaffolding despite their hacky origins as formatting tricks
- Understanding and improving role perception is crucial for security against prompt injection and controlling LLM behavior
