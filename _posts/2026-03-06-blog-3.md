## Key Points

- This is a research paper titled 'Agentic Code Reasoning' exploring whether LLM agents can analyze codebases and understand code semantics without executing the code
- The authors introduce 'semi-formal reasoning' - a structured prompting methodology that requires agents to construct explicit premises, trace execution paths, and derive formal conclusions
- Unlike unstructured chain-of-thought prompting, semi-formal reasoning acts as a certificate that prevents agents from skipping cases or making unsupported claims
- The methodology was evaluated across three tasks: patch equivalence verification, fault localization, and code question answering
- For patch equivalence verification, accuracy improved from 78% to 88% on curated examples and reached 93% on real-world agent-generated patches
- For code question answering on RubberDuckBench, semi-formal reasoning achieved 87% accuracy
- For fault localization on Defects4J, semi-formal reasoning improved Top-5 accuracy by 5 percentage points over standard reasoning
- These results show that structured agentic reasoning enables meaningful semantic code analysis without execution
- The approach has practical applications in RL training pipelines, code review, and static program analysis
