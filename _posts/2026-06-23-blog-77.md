## Key Points

- Security researchers have published a working exploit (CVE-2026-23111) for a Linux kernel use-after-free vulnerability in the nf_tables packet-filtering code that allows unprivileged local users to escalate to root privileges
- The flaw originated from a single stray character - an inverted check in nf_tables - and was fixed upstream with just one line of code removal on February 5, 2026
- The exploit enables attackers to break out of containers and requires two common Linux features: nf_tables plus unprivileged user namespaces, both of which are enabled by default on most desktop and many server installations
- Two independent research teams have published exploits: Exodus Intelligence released their technical walkthrough on June 8, while FuzzingLabs had already published a reproduction in April
- The vulnerability has been successfully demonstrated on multiple distributions including Debian Bookworm, Debian Trixie, Ubuntu 22.04 LTS, and Ubuntu 24.04 LTS, with Ubuntu rating the flaw CVSS 7.8 (high)
- There is no remote attack vector - this is a local-only bug that attackers can use after gaining initial foothold to escalate privileges from low-privileged shells, compromised containers, or service accounts
- The bug is part of a recent surge of Linux local privilege escalation (LPE) disclosures including Copy Fail, Dirty Frag chain, Fragnesia variant, DirtyDecrypt, and a nine-year-old ptrace flaw
- Major distributions have released patches: Ubuntu for 22.04, 24.04, and 25.10; Debian for Bookworm, Trixie, and Bullseye LTS; with Red Hat, SUSE, and Amazon Linux also tracking the flaw
- Researchers note no public reports of in-the-wild exploitation, with the patch being available since February and exploit code public since April
- The exploit requires unprivileged user namespaces, so defenders are advised to disable or restrict this feature on systems where untrusted users or workloads could create them
