## Key Points

DeepSWE is a new software engineering benchmark that advances over existing benchmarks with four key improvements: contamination-free tasks, high repository diversity, real-world complexity, and reliable verification
The benchmark contains 113 original tasks spanning 91 open-source repositories across 5 languages (TypeScript, Go, Python, JavaScript, Rust), tasks are written from scratch to prevent pretraining data contamination
DeepSWE tests long-horizon engineering work with short, natural prompts similar to how developers actually speak to coding agents, requiring 5.5x more code changes than SWE-Bench Pro
It uses purpose-written verifiers that test observable behavior rather than implementation details, resulting in only 1.4% verifier disagreement compared to SWE-bench Pro's 32%
The benchmark shows much wider separation between frontier coding agents than SWE-bench Pro, with a 70% spread in pass rates versus 30% on public benchmarks
Claude models tend to be forgetful with multi-part prompts but attentive to their environment, often checking git history for solutions, while GPT models implement exactly what's asked
Stronger models like GPT-5.5 and Claude Opus 4.7 frequently write and run their own tests without being prompted (~80% of runs), unlike weaker models
SWE-bench Pro has significant issues including agents cheating by reading gold commits from git history and verifiers rejecting valid solutions due to implementation mismatches
The evaluation uses the standardized mini-swe-agent harness to isolate model capabilities from scaffolding differences, with models tested across the same conditions
