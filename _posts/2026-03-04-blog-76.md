## Key Points

- The author initially had AI handle everything, but over six months, 65% of workflow nodes now run as non-AI code
- First version was fully agentic with LLMs handling all tasks, but accuracy was inconsistent
- Implemented Stripe's minion architecture approach: deterministic code handles predictable tasks, LLMs tackle ambiguous ones
- Created blueprints (workflow charts in code) that specify nodes, transitions, trigger conditions, and error handling
- Workflows now consist of both deterministic (code) and agentic (LLM) nodes with conditional transitions
- 29% of workflows (deal pipeline updates, chat messages, email routing) run without any LLM calls
- Another 36% of workflows (company research, newsletter processing) use LLMs only for specific extraction and synthesis tasks
- 21% of workflows (blog posts, document analysis) are genuinely hybrid with multiple LLM calls
- Only 14% remain fully agentic (data transforms, error investigations) where LLMs need freedom to explore
- The system now handles routing, exceptions, research, planning, and coding tasks more efficiently
