## Key Points

When training LLMs with reinforcement learning (RL) for agentic behavior using tools, most implementations break the Token-In, Token-Out (TITO) invariant.
The core problem is that re-tokenizing the conversation after tool calls can create different token sequences from what the model originally generated, leading to broken gradients.
The solution is a simple rule: never re-encode tokens you've decoded - maintain a token buffer and only append tool responses as deltas.
Most modern chat templates (18 of 19 tested) are prefix-preserving for tool messages, meaning appending tool results doesn't break existing tokens.
The only model that needs a fix is Qwen3, which can be corrected with a one-line change to its Jinja template.
This approach is simpler than using renderer libraries that require per-model implementations and maintenance.
The method handles edge cases like history rewriting and truncation more gracefully than renderer-based approaches.
The key insight is that you only need token-level control, not string-level control, for correct RL training of tool-using agents.
