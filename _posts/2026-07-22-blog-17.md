## Key Points

- Netflix built its own in-house LLM serving platform rather than using hosted APIs, running the full stack from model deployment through inference in their existing production environment
- They use vLLM as their primary inference engine due to its custom model architecture support, extensibility for custom decoding logic, better debuggability, and team familiarity
- The system integrates vLLM with NVIDIA Triton Inference Server, using Triton's vLLM backend to decouple model artifacts from frontend upgrades
- Netflix exposes an OpenAI-compatible API alongside their existing gRPC interface, making it easier for teams to transition from hosted models to self-hosted ones
- They offer two deployment strategies: Red-Black for stable interfaces and Versioned for breaking interface changes, each with different trade-offs for coordination and cost
- Production revealed unexpected challenges like Triton/vLLM version mismatches that can cause complete backend failures if not properly pinned
- Netflix implements constrained decoding at scale using vLLM's custom logits processors, which evolved from a CPU-bound per-request implementation to a more efficient batch-level design in vLLM V1
- The platform addresses operational issues like model caching for faster cold starts and unified metrics collection from both vLLM and Triton
- Next investments include system prompt compression, asynchronous scheduling, vectorized logits processors, and lower-precision model variants
