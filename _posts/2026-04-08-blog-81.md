## Key Points

- Reinforcement learning (RL) environments are critical but often overlooked components that determine what LLM agents can learn to do, comprising tasks, agent harness, verifier, state, and configuration
- Task types vary from single-turn Q&A to complex enterprise workflows, coding environments, and open-ended research, with proper task distribution being essential for agent capability development
- The agent harness controls how models interact with environments, including rollout protocols (single-turn to full agentic), tools, context management, and system prompts
- Verifiers map agent completions to rewards, with programmatic verification being preferable to LLM-as-judge when possible, and evolving rubrics helping prevent reward hacking
- Environments can be stateful or stateless, with enterprise environments requiring persistent state across actions and episodes
- Benchmarks are essentially frozen RL environments, sharing the same components but without the ability to evolve during training
- Recent trends include automated environment generation ($4 per environment), managed environment services (Prime Intellect Environments Hub, OpenReward), and the emergence of standard protocols like ORS (Open Reward Standard)
- Environment diversity is as important as quality, with heterogeneous environments driving broader capabilities in agents
- Contamination resistance and dynamic task generation are becoming design requirements to prevent models from memorizing benchmark answers
