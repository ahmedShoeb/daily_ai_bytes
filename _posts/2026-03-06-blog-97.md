## Key Points

Cursor support engineers use Cursor itself for over 75% of support interactions, increasing throughput 5-10x
Investigations start in Ask Mode, tracing symptoms backward through the codebase with full product code, docs, and internal tools available locally
Multi-root workspaces allow linking related repositories to answer complex questions spanning frontend, backend, and documentation
MCP servers integrate databases, event logs, Slack, ticket platforms, documentation services, and account management systems directly into investigations
Datadog MCP helps identify where failures occurred by pulling relevant logs and traces into investigation threads
Support teams use MCP to search previous cases across platforms like support systems and Slack for similar issues and solutions
Notion MCP helps determine if issues are bugs or expected behavior by cross-referencing against runbooks
Linear MCP enables creating formatted bug reports directly from investigation threads
Support engineers can implement documentation fixes themselves using cloud agents that open PRs against docs repos
The team uses slash commands, Rules, Skills, and subagents (LogInvestigator, KnownIssueMiner, TicketWriter, CustomerReplyDrafter) to automate common support processes
This AI-native approach makes support engineers up to an order of magnitude more productive compared to traditional methods
