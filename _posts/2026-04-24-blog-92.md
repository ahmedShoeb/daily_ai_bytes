## Key Points

WorkOS has launched a CLI tool that provides one-command AuthKit integration, going from zero authentication to fully integrated in about two minutes
The CLI's headline feature is an AI Installer that auto-detects frameworks, installs SDKs, creates routes, sets up environments, and validates builds automatically
It supports a wide range of frameworks including Next.js, React, SvelteKit, Node.js, Python, Ruby, Go, PHP, .NET, Kotlin, and Elixir
You can install WorkOS knowledge directly into AI coding agents like Claude Code, Codex, Cursor, and Goose to help them produce correct AuthKit integration code
The CLI provides full CRUD operations for WorkOS resources (organizations, users, roles, permissions, directories, etc.) from the terminal
It supports environment management with production, staging, and local environments that you can switch between easily
There's a 'doctor' command that diagnoses integration issues by checking SDK versions, environment configuration, connectivity, and auth patterns
The tool includes seed commands that allow you to define permissions, roles, and organizations in YAML and provision them with one command
Debug commands are available for inspecting SSO connection states, directory sync states, and troubleshooting complex issues
All resource commands support JSON output for scripting and CI pipelines, with automatic JSON output in non-TTY environments
