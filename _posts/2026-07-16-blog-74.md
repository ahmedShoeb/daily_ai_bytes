## Key Points

-Security researcher reveals xAI's Grok Build CLI transmits entire code repositories to xAI servers without user disclosure
-Grok uploads all tracked files including .env secrets files verbatim and unredacted to Google Cloud Storage bucket grok-code-session-traces
-The upload occurs via two channels: model-turn channel for files actually read, and whole-repository channel that uploads everything
-In tests, Grok uploaded 5.1 GB of a 12 GB repository containing only random files that were never read by the AI
-Even when users disable Improve the model setting, the codebase uploads continue and server returns trace_upload_enabled: true
-Researchers captured git bundle uploads containing files the AI was explicitly told not to read, proving entire repository transfer
-The upload mechanism is active by default on standard consumer accounts and not documented in CLI setup materials
-No storage request failures were observed in testing - all /v1/storage calls returned HTTP 200 status
-The behavior is version-specific to grok 0.2.93 from July 2026
-Security implications include potential exposure of API keys, passwords, and proprietary code without user knowledge
