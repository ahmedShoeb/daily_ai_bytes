## Key Points

Miles is RadixArk's open-source reinforcement learning framework designed for large-scale LLM post-training, built to address the distributed systems challenges of modern RL
The framework composes four major systems: SGLang for high-throughput rollout generation, NVIDIA Megatron-LM for scalable distributed training, Ray for cluster orchestration, and PyTorch as the common programming layer
Miles handles the unique challenges of RL post-training, including coordination between rollout and training phases, weight synchronization, and maintaining alignment for mixture-of-experts (MoE) models
The system uses Ray's distributed runtime for orchestrating long-running RL jobs, with GPU-aware scheduling, fault tolerance, and support for fully asynchronous execution
It integrates deeply with Megatron-LM for frontier-scale training, supporting both dense and MoE models while maintaining a clean user-facing workflow
Miles provides PyTorch-native extensibility, allowing users to customize models, rewards, losses, and algorithms without learning new abstractions
Key out-of-the-box features include MoE-aware rollout/training alignment, fast NCCL/RDMA weight synchronization, unified low-precision recipes (BF16/FP8/MXFP8/INT4-QAT), and LoRA support across both rollout and training
The framework follows a 'small-core, many-edges' philosophy with explicit extension points for rollout functions, reward functions, loss functions, sample filters, training hooks, and model specs
Miles ships with ready-to-run recipes for frontier models including DeepSeek-V4, Kimi K2.5/K2.6, GLM-5/5.1, and Qwen3.5/3.6, with support for NVIDIA Hopper/Blackwell GPUs
The goal is to make frontier-scale LLM RL training more composable, reproducible, and easier to scale while keeping customization accessible to researchers and infrastructure teams
