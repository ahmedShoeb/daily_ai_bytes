## Key Points

OpenAI is extending Codex's remote control system to let the coding agent operate macOS applications through Computer Use even when a laptop is locked or asleep
This addresses a major workflow gap where Computer Use previously required an unlocked, awake session to see the screen, move the cursor, and type in apps
The capability would allow phones to direct Codex to open desktop apps, test GUI builds, run simulators, or access data sources without users needing to physically log into their machines
The development closes a gap with Anthropic, which shipped similar phone-to-machine control for Claude Code in February but also faces the locked session constraint
OpenAI is exploring connecting to and controlling other desktop devices running the Codex app, such as installing it on a Mac Mini and operating it from a main device
Users will potentially be able to connect to and operate many devices remotely based on developing UI components
This development raises security questions as it bypasses macOS's standard expectation that a locked screen means an untouchable session and could draw attention from Apple
The release timing hasn't been announced but should be viewed as part of OpenAI's larger remote-control strategy rather than a standalone launch
