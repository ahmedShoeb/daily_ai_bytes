## Key Points

- CUDA kernels go through multiple compilation stages: from CUDA C++ to PTX (virtual ISA) to SASS (actual GPU machine code), all bundled into a single executable
- The host launch process involves setting up arguments in constant memory, using a pushbuffer/GPFIFO system to queue work, and ringing a 'doorbell' register to notify the GPU
- GPU execution uses a complex scheduling system where warps are managed across Streaming Multiprocessors (SMs) with compiler-written stall counts and hardware scoreboards
- Memory accesses are coalesced at the warp level - consecutive float reads become single 128-byte requests to maximize memory bandwidth efficiency
- The GPU hides latency not through out-of-order execution like CPUs, but by keeping many warps resident and switching between them when they stall
- The entire launch process from CUDA code to GPU execution involves millions of CPU instructions, hundreds of ioctl calls, and a single memory-mapped doorbell register
- Modern NVIDIA GPUs (Ada architecture) use compiler-generated control codes in the SASS instructions to schedule operations with minimal hardware overhead
- The kernel execution path demonstrates how a simple vector addition involves coordination across PCIe, driver communication, and complex GPU hardware management
