## Key Points

SRAM-centric AI accelerators like Cerebras, Groq, and d-Matrix are gaining traction with major deals including NVIDIA licensing Groq's IP for $20B and Cerebras announcing a 750 MW deal for OpenAI inference workloads
These chips prioritize near-compute memory (SRAM) over far-compute memory (HBM/DRAM), offering significant latency and throughput advantages for inference workloads
SRAM is faster than HBM due to two main factors: SRAM reads are physically faster, and SRAM lives on-chip with compute cores while HBM lives off-chip
GPUs handle memory scaling challenges through caching and data reuse, while SRAM-centric architectures sacrifice compute capacity for larger on-chip memory pools
Arithmetic intensity determines which architecture wins - GPUs excel at high compute intensity workloads like training and prefill, while SRAM-centric chips excel at memory bandwidth-bound workloads like decode
Decode (token-by-token generation) is particularly problematic for GPUs because it's highly auto-regressive, forcing frequent weight reloading and creating memory bottlenecks
The article predicts disaggregation will become standard, with prefill running on compute-heavy hardware and decode running on memory-heavy hardware like SRAM-centric chips
Gimlet Labs has built a multi-silicon inference cloud that intelligently splits workloads across different hardware types, mapping different phases to optimal accelerators
The industry is moving toward phase-specialized inference hardware, with both NVIDIA's Rubin CPX (using GDDR7 instead of HBM) and d-Matrix's Raptor chip (stacked DRAM) representing new approaches
Memory density challenges continue as SRAM cell size and DRAM cost per byte have stalled while compute continues scaling, creating pressure for new memory architectures
