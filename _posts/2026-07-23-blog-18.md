## Key Points

- Ramp built an internal LLM gateway processing trillions of tokens daily serving all their AI use cases
- They implemented a dynamic failure-aware routing strategy using Thompson sampling that reduced AI spend by over 25% while also lowering error rates
- The system learns real-time latency and failure distributions online using a Normal-Inverse-Gamma prior on unknown mean and variance of log-latency
- Routing decisions combine failure probability (from EWMA) with latency probability and cost to score each model option
- The strategy dynamically reroutes traffic based on real-time performance, such as avoiding rate-limited models like Gemini 3.1 Flash Lite during outages
- Ramp deployed the system first to their LLM reranker use case with a 6-second deadline, then expanded to streaming use cases
- During high traffic periods, the system automatically selected cheaper flex tiers when performance was comparable to standard tiers
- They've since implemented a streaming-equivalent version for Ramp Inspect, resulting in 30% cost savings
- The centralized gateway approach enables organized spend attribution and optimization opportunities
