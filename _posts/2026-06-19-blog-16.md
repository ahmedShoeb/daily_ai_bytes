## Key Points

- Microsoft is considering extending Phi Silica GPU support to supported Nvidia GPUs for local AI models, potentially expanding local AI beyond Copilot+ PCs
- Developers need specific requirements: Experimental Windows Insider channel, Developer Mode, Windows App SDK 2.2.2-experimental9 or later, and current GPU drivers
- Hardware requirements include RTX 30-series or newer GPUs with at least 6GB VRAM, with AMD support planned for later
- GPU execution lacks some NPU-only capabilities like prompt compression and speculative decoding, affecting context handling and generation speed
- Phi Silica is derived from Phi-3 architecture and designed as Small Language Models for low-latency local processing
- Apps must check GetReadyState flag before invoking models to avoid presenting GPU support as available prematurely
- Microsoft's Windows AI strategy remains fragmented by model, API, GPU vendor, and release channel
- No broad consumer-release schedule has been provided for the GPU path; it remains in developer/testing phase
- This approach gives Microsoft a way to reach developers who already have supported graphics hardware without changing NPU-focused Copilot+ PC category
