## Key Points

Ossature is an open-source harness for spec-driven code generation that addresses the challenge of getting LLMs to produce coherent multi-component software projects.
The tool was created in response to observations from OpenAI and industry experts about the need for structured environments (harnesses) that keep AI-generated codebases coherent and maintainable.
Users write specifications in a Markdown-based format called SMD that describes concrete behavior, interfaces, dependencies, and error handling - with more detail leading to better LLM implementation.
Ossature operates in three stages: validate (parsing and dependency checking), audit (LLM semantic review for ambiguities/gaps), and build (executing generation tasks).
The system generates a human-editable build plan (TOML file) that breaks down tasks with dependencies, context files, and verification commands for each step.
During build, each task gets only the necessary context (spec sections, interface files from dependencies, output from earlier tasks) rather than full project dumps, reducing confusion and architectural drift.
Includes built-in verification loops where each task runs a verification command after generation, with fixer agents attempting repairs up to 3 times if verification fails.
Builds are incremental - only affected tasks rebuild when specs change, and all prompts/responses are logged for debugging when issues arise.
The project is at version 0.0.1 (considered unstable), MIT-licensed, and supports major LLM providers including Anthropic, OpenAI, Mistral, Google, and local models through Ollama.
Currently optimized for greenfield projects but the team is working on support for existing codebases to make spec-driven generation work incrementally on established systems.
