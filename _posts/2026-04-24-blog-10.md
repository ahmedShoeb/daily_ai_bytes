## Key Points

WorkOS launched 'Agent Experience' - a set of tools designed for AI coding agents to work without human dashboard interaction
Six major upgrades: WorkOS Skills for better agent understanding, try AuthKit without account creation, workos doctor for diagnostics, dashboard-free configuration, declarative environment setup with workos seed, and resource commands for data access
WorkOS Skills provide agents with baseline understanding of WorkOS APIs, concepts, and integration steps without manual prompt engineering
You can now integrate AuthKit without creating a WorkOS account first - WorkOS creates an unclaimed environment for testing
workos doctor command inspects projects and flags common misconfigurations like redirect URI mismatches
CLI can now handle configuration tasks like setting redirect URIs and managing webhooks without dashboard access
Declarative environment setup with workos seed allows defining WorkOS environments as code in YAML files
New resource commands give agents programmatic access to roles, permissions, audit logs, directories, and users
Designed for developers using AI agents like Claude, Cursor, or Codex in their workflow
Transforms development workflow from dashboard-dependent to terminal-focused agent-driven development
