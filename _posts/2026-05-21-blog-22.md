## Key Points

- Extends Andrej Karpathy's original LLM Wiki concept with lessons from building agentmemory, a persistent memory engine for AI agents
- Proposes adding confidence scoring to wiki facts to track how many sources support them, how recently confirmed, and whether contradictions exist
- Introduces supersession concept where new information explicitly replaces old claims with version control and timestamps
- Suggests implementing forgetting mechanisms using retention curves based on Ebbinghaus's forgetting curve for knowledge
- Describes four-tier consolidation pipeline: working memory, episodic memory, semantic memory, and procedural memory
- Recommends building a typed knowledge graph on top of wiki pages with entity extraction and typed relationships
- Proposes hybrid search combining BM25 keyword matching, vector search for semantic similarity, and graph traversal
- Outlines automation hooks for events like new source ingestion, session starts/ends, and periodic lint operations
- Emphasizes quality controls including scoring mechanisms, self-healing capabilities, and contradiction resolution
- Discusses collaboration features for multi-agent systems including mesh sync between agents and private vs. shared knowledge
