## Key Points

- Microsoft tested whether rewriting CLI tools to use JSON instead of traditional command-line arguments would improve agent (AI) performance
- They created a synthetic CLI called 'podctl' with complex deployment scenarios involving multiple services and nested configurations
- Traditional CLI arguments achieved perfect correctness across all tested models, including smaller models like Claude Haiku
- JSON mode had lower correctness with smaller models (Haiku got 2/5 correct, MAI-Code-1-Flash got 3/5)
- JSON mode cost 4x to 11x more per task due to increased retries and token usage
- Shell escaping was a major problem with JSON - valid JSON would break due to shell quoting differences (PowerShell vs Bash)
- Traditional args work across all model capability levels and don't have environment-dependent failure modes
- The constraints of traditional args help agents by narrowing the valid input space and eliminating complexity
- The study advises against removing CLI arguments in favor of JSON for agent usage
- Recommendation: Measure before rewriting CLI tools and keep traditional args for agent compatibility
