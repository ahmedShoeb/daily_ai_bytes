## Key Points

- When LLM inference streams get interrupted (due to crashes, deploys, etc.), you not only lose progress but also waste money by paying for the same output tokens multiple times
- The core problem is that provider connections live inside the processes that crash - when those processes die, you have to make the calls again and pay for tokens you've already been billed for
- Solution: Move provider connections to a separate durable buffer deployment (like a Durable Object) that outlives your agent processes
- This durable buffer stores stream chunks in SQLite as they're generated, allowing recovery from where you left off instead of starting over
- The same mechanism enables both resumable streaming (for disconnected clients) and crash recovery - both use the same durable chunk log
- OpenAI's Responses API already has background mode with resume capability, but most other providers (Anthropic, Gemini) require re-prompting and re-paying
- Cloudflare AI Gateway is adding durable resume as a first-class feature, allowing buffer-free crash recovery across all providers
- Using providers' native SSE parsers avoids maintaining custom parsers and keeps you compatible with their format changes
- Durable inference saves significant money, especially with expensive models where retries cost 15x more than with cheaper alternatives
- The author implemented this as a proof-of-concept Durable Object but argues the feature belongs in managed infrastructure that never redeploys
