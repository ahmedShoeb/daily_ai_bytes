## Key Points

- lat.md is a knowledge graph system that organizes codebase documentation into interconnected markdown files instead of monolithic documents
- It creates a lat.md/ directory at the root of projects where markdown files describe architecture, business logic, and test specs
- The system uses [[wiki links]] for section-to-section connections and links to source code files
- Code can link back to the knowledge graph using // @lat: [[section-id]] comments
- The lat CLI provides commands like 'lat init' (setup), 'lat check' (consistency validation), and 'lat search' (semantic search)
- Key benefits include: faster agent coding through improved context discovery, better human workflow with semantic change review, knowledge retention across sessions, and test spec enforcement
- Test specs can be marked with require-code-mention: true to ensure proper test coverage maintenance
- The system supports semantic search through OpenAI or Vercel AI Gateway APIs
- Installation is available via npm with 'npm install -g lat.md'
