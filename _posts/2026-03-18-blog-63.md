## Key Points

Language models can solve complex math problems but struggle with basic computational tasks that require reasoning over many steps
The researchers built a computer inside a transformer that can execute arbitrary C code directly, enabling millions of execution steps in seconds
Standard LLMs rely on external tools for computation, but this approach enables internal execution where the model itself runs programs step by step
The key innovation is a new decoding path that transforms attention lookups from linear scans into logarithmic-time queries using 2D attention heads
The system can execute a WebAssembly interpreter inside transformer weights, producing execution traces as tokens without external round-trips
This approach enables solving notoriously hard problems like the Arto Inkala Sudoku with 100% accuracy through compiled solver execution
The fast path requires 2D attention heads which enable geometric optimization using convex hull queries instead of brute-force scanning
The method achieves exponential speedup - O(log n) instead of O(n) per step - making million-step execution traces feasible
The transformer architecture remains standard PyTorch, with the only special requirement being 2D heads for the fast execution path
This research opens possibilities for AI systems that integrate learned representations with compiled algorithms as part of their internal architecture
