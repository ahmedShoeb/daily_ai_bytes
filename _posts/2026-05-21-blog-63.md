## Key Points

- Lovable has introduced Skills, a system that allows users to create reusable instructions for AI tasks
- Skills solve the problem of constantly re-explaining preferences, conventions, and workflows to AI assistants
- The format originated from Anthropic and has been adopted by Lovable, OpenAI, and others
- Skills are markdown files that are portable, readable, and editable, making them easy to share between tools
- Each skill has three components: name, description, and instructions, with description being critical for when the AI decides to use it
- Skills are on-demand, only loading when relevant based on their description, and multiple skills can fire on the same task
- There are real-world examples provided including design-system, fresh-eyes-review, and landing-page-copy skills
- The article contrasts good vs bad skill descriptions with specific examples showing what makes descriptions effective
- Skills have honest caveats: they're only as good as what's in them, don't replace prompting, and require maintenance
- Practical guidance is included on avoiding overlapping skills, dealing with conflicts, and getting started with built-in templates
