## Key Points

Pretext is a TypeScript text layout algorithm that reveals better patterns for using AI coding agents on messy, empirical problems
The key insight: Lock down architecture constraints first so AI has less room to make poor decisions
Use real-world measurements (browser behavior) as an 'oracle' to validate AI suggestions rather than trusting AI as authority
Shrink broad failures into tiny, reproducible probes before attempting solutions
Name failure modes to create taxonomies that help distinguish different types of bugs
Use AI for throughput (building probes, running checks, testing hypotheses) rather than as the decision-making authority
Expect to reject most AI-generated patches - high rejection rates are healthy in rigorous engineering
Build layered validation stacks with multiple test types to avoid model overfitting or missing regressions
The effective AI coding loop: constrain → measure → isolate → classify → test → reject → keep only what survives pressure
AI doesn't make engineering rigorous - the disciplined process and validation loop does
