## Key Points

- Claude Code is introducing a new auto mode feature that allows Claude to make permission decisions on behalf of users
- Auto mode is a middle path between conservative default permissions and skipping all permissions with --dangerously-skip-permissions
- A classifier reviews each tool call for potentially destructive actions before they run, blocking risky actions and allowing safe ones to proceed automatically
- Actions that may be blocked include mass deleting files, sensitive data exfiltration, or malicious code execution
- If Claude insists on repeatedly blocked actions, it will eventually trigger a permission prompt to the user
- Auto mode reduces risk compared to skipping all permissions but doesn't eliminate risk entirely
- The feature is available now as a research preview for Team plan users
- It will roll out to Enterprise plan and API users in the coming days
- Auto mode works with both Claude Sonnet 4.6 and Opus 4.6 models
- There may be a small impact on token consumption, cost, and latency for tool calls
