## Key Points

The Model Context Protocol (MCP) 2026-07-28 release candidate introduces the largest protocol revision since launch with a stateless core that eliminates the need for sticky sessions and shared session stores
MCP is now stateless at the protocol layer - the initialize/initialized handshake and Mcp-Session-Id headers are removed, allowing any server instance to handle any request
The stateless design enables production deployments to run behind simple round-robin load balancers and cache tool list responses, simplifying infrastructure requirements
Two official extensions are introduced: MCP Apps for server-rendered HTML interfaces in sandboxed iframes, and Tasks extension with a redesigned stateless lifecycle
Authorization is hardened with six SEPs aligning more closely with OAuth 2.0 and OpenID Connect deployments, including mandatory issuer validation and better client registration
Three core features are deprecated under the new lifecycle policy: Roots, Sampling, and Logging - with clear replacements documented for each
Tool schemas are upgraded to full JSON Schema 2020-12 support, allowing composition, conditionals, and references in input and output schemas
The protocol now includes proper load balancer support with Mcp-Method and Mcp-Name headers, cache control via ttlMs and cacheScope, and standardized W3C trace context propagation
A formal extension framework is established with reverse-DNS IDs, extension repositories, and separate versioning from the core specification
The release candidate is available now, with the final specification scheduled for July 28, 2026, and includes breaking changes that require migration from the 2025-11-25 version
