## Key Points

- The article presents a multi-agent harness design inspired by Generative Adversarial Networks (GANs) for improving Claude's performance on long-running applications
- Key innovation: Separates generator and evaluator agents to overcome AI self-evaluation bias where models tend to praise their own work even when quality is mediocre
- For frontend design, developed grading criteria to turn subjective judgments into concrete, gradable terms, with the evaluator using Playwright MCP to interact with live pages
- Applied the approach to full-stack development with a three-agent system: planner, generator, and evaluator agents working together
- The harness addresses 'context anxiety' where models lose coherence on lengthy tasks or wrap up work prematurely as context limits approach
- Initial tests showed dramatic improvement: A retro game maker built with the harness was 20x more expensive but produced functional, polished applications versus solo runs
- With Claude Opus 4.6, the team simplified the harness by removing sprint constructs while keeping planner and evaluator agents
- A Digital Audio Workstation (DAW) example demonstrated the system could create functional, AI-integrated applications through multi-hour autonomous coding sessions
- The work shows that as models improve, harness design needs to adapt - removing scaffolding when models can handle tasks solo, but adding complexity for more advanced capabilities
- Key insight: Harness design creates a space of interesting combinations that doesn't shrink as models improve, but instead moves to enable more complex tasks
