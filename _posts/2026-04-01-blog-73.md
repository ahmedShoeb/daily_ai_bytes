## Key Points

AutoBe is an AI agent that generates production-grade backends from natural language using function calling, achieving 99.8%+ compilation success across five Qwen models
Initial function calling success rate was only 6.75% when using qwen3-coder-next to generate API data types for complex schemas
The breakthrough came from building a 'harness' - type schemas that constrain outputs, compilers that verify results, and structured feedback that pinpoints errors
Typia is a key infrastructure that automatically generates JSON Schema, parsers, validators, and feedback from TypeScript types, enabling lenient JSON parsing and type coercion
Function calling's advantage over natural language prompts is its ability to make unwanted outputs structurally impossible through schema constraints rather than prohibitions
The approach is model-neutral - same schema works across Qwen, GLM, DeepSeek, and OpenAI models, changing model selection from capability assessment to cost optimization
The pattern applies to any domain with deterministic validators - semiconductors, chemical processes, construction, control systems - not just software development
Small models like Qwen's 3B active parameter models are valuable QA engineers that expose system vulnerabilities more effectively than large models that 'correctly guess' ambiguous parts
The self-healing loop combines parse (broken JSON recovery), validate (schema checking), and structured feedback to turn probabilistic model output into deterministic results
The core insight: 'If you can verify, you converge' - deterministic validation loops around probabilistic models enable 100% success rates from any starting point
