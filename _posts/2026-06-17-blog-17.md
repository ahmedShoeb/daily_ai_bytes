## Key Points

1. The article provides a framework for estimating AI inference costs using 'napkin math' - back-of-the-envelope calculations for scaling AI model services.
2. It focuses on key metrics for cost estimation: GPU memory bandwidth, compute throughput, model size (32B parameters), and context length (200k tokens).
3. The core insight shows that matrix multiplication operations in LLMs require far more compute operations than memory accesses - four orders of magnitude more.
4. KV-caching (key-value caching) is identified as crucial optimization that reduces computation by caching previously processed tokens rather than recalculating entire chat histories.
5. Using NVIDIA B200 GPU as an example: 8 TB/s memory bandwidth and 4500 TFLOP/s compute, allowing 562 computations per byte loaded for optimal efficiency.
6. Realistic GPU utilization shows about 6 concurrent users at 100% duty cycle due to VRAM limitations for weights (32GB) plus KV-cache requirements.
7. Through optimization techniques like Grouped-Query-Attention and PagedAttention (vLLM), user capacity increases dramatically to 300 800 users per GPU.
8. Performance calculations show ~40 tokens per second per user for 6 concurrent users, which exceeds most reading speeds.
9. Cost analysis shows worst-case lifetime cost of $6,000 per user at 100% duty cycle, but realistically ~$133 per user for 500 users per GPU.
10. Cloud rental economics: $3/hour for B200 translates to ~$4.32/month per user for 500 users, making it profitable to charge above this threshold.
