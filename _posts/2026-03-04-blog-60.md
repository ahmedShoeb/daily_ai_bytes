## Key Points

- Author makes a bold claim that MCP (Model Context Protocol) is already dying despite initial industry excitement
- Notes that major AI platforms like OpenClaw and Pi don't support MCP, suggesting this isn't accidental
- Argues LLMs are already great at using command-line tools without needing special protocols
- Highlights that CLIs are better for debugging since humans can run the same commands and see the same output
- Emphasizes CLI composability advantage - ability to pipe, chain, and redirect output using existing tools
- Points out that CLI authentication systems are already battle-tested and work the same for humans and agents
- Criticizes MCP's practical issues: flaky initialization, repetitive re-authentication, and all-or-nothing permissions
- Acknowledges MCP might make sense for tools with no CLI equivalent, but claims CLI is better for most work
- Suggests companies should focus on building good APIs and CLIs rather than MCP servers, since agents will figure it out
