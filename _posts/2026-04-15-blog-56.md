## Key Points

- This research paper presents the first systematic study of attacks on LLM API routers in the supply chain
- Researchers formalized a threat model with two core attack classes: payload injection (AC-1) and secret exfiltration (AC-2)
- Investigated 28 paid routers from marketplaces and 400 free routers, finding active malicious code injection in 1 paid and 8 free routers
- Discovered adaptive evasion variants including dependency-targeted injection (AC-1.a) and conditional delivery (AC-1.b)
- Found routers touching researcher-owned AWS credentials and one draining ETH from a private key
- Poisoning studies showed that apparently benign routers could be compromised to generate 100M GPT-5.4 tokens and access 99 credentials across 440 Codex sessions
- Built a research proxy called 'Mine' that implements attack classes against four public agent frameworks
- Evaluated three client-side defenses: fail-closed policy gate, response-side anomaly screening, and append-only transparency logging
- Demonstrates significant security vulnerabilities in third-party API routers used by LLM agents with full plaintext access to JSON payloads
- Highlights that no provider currently enforces cryptographic integrity between clients and upstream models
