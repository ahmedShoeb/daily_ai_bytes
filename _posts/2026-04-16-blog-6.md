## Key Points

- The author describes using AI agents as 'scaffolding' for automating recurring tasks, using security vulnerability patching as a concrete example
- Initially created an agent to handle Github Dependabot webhooks for security alerts, but encountered reliability issues with severity filtering
- Found that agents as drop-in software replacements often lack the perfection needed for interrupting colleagues with notifications
- Developed a hybrid approach: using code-driven workflows for deterministic tasks and reserving agents for ambiguous problems like identifying code owners
- Current workflow: webhooks → code filtering → agent identifies owners → second agent formats Slack messages → human review where needed
- This pattern of prototyping with agents, then refactoring to code-driven control while keeping agents for their strengths has worked well across many problems
- The approach results in faster, cheaper, and more maintainable solutions
- Future improvements include automated vulnerability fix generation and weekly follow-ups for open critical issues
- The author emphasizes that this transition is relatively cheap and quick using tools like Claude/Codex
