## Key Points

- Compound Engineering (CE) introduces a verification loop where agents autonomously test features before deployment, addressing the challenge of ensuring code actually works in production
- The /ce-dogfood skill acts as a QA engineer that tests every change in a real browser, using the agent-browser CLI to drive browser interactions directly
- The approach is diff-scoped - only testing what changed versus the trunk, not the entire application
- Dogfood focuses on user flows rather than isolated pages, mapping every user-visible change as explicit journeys through the application
- Quality is judged on two levels: functional correctness (does it work) and experiential quality (does it feel right) using product personas
- The fix loop includes safeguards - only obvious bugs are auto-fixed with accompanying regression tests, while complex issues get escalated to humans
- The system uses a persona strategy with seeded specialists rather than standing agents, keeping skills self-contained and avoiding stale registrations
- Every verification step leaves an audit trail, with reports documenting scenarios, fixes, and decisions that need human review
- The approach prevents silent failures by making open questions visible through 'Blocked' states that must be explicitly addressed
- Verification loops exist at multiple altitudes from unit tests to end-to-end pipelines, all following the same claim-plus-proof grammar
