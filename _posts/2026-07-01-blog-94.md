## Key Points

- The article proposes researching how coding LLM perplexity scales with codebase size across different programming languages
- Suggests measuring LLM predictability to estimate scaling laws, which could reveal which languages become easier for AI to work with at scale
- Hypothesizes that Lean programming language has worse baseline performance but better scaling exponents than popular languages like Python
- Suggests that strongly typed languages like Lean may eventually outperform dynamic languages at large codebase scales despite initial disadvantages
- Proposes using frozen LLMs to measure perplexity across codebases and compare predictability scaling exponents between languages
- Predicts a 'crossover' point where Lean becomes more predictable than popular languages at certain codebase sizes
- Acknowledges potential biases in research due to different programming communities and application domains
- Suggests this research could motivate investments in Lean development and codebase rewrites for improved software safety
