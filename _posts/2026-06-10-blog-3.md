## Key Points

Explains how modern transformer-based LLMs work by breaking down their core components: tokenization, embeddings, positional encoding, attention mechanisms, and more.
Tokenization converts text into integer IDs using subword pieces (not whole words) for efficiency, with different tokenizers like Byte Pair Encoding and SentencePiece used across LLM families.
Embeddings transform token IDs into vectors that capture semantic meaning, where similar meanings have similar vector positions, enabling arithmetic like 'king - man + woman ≈ queen'.
Positional encoding, particularly Rotary Position Embeddings (RoPE), helps models understand word order by rotating Query and Key vectors based on token position, addressing sequence awareness issues.
Attention mechanisms allow tokens to exchange information through Query, Key, and Value vectors, with causal masking preventing future token viewing in decoder-only models.
Multi-head attention runs parallel attention passes to capture different types of relationships, with modern models using Grouped-Query Attention (GQA) to reduce memory costs.
Feed-forward networks process each token independently, housing most of a model's parameters and forming the basis for factual and conceptual memory storage.
Advanced techniques include Mixture of Experts (MoE) for efficient large models, where only select feed-forward networks activate per token.
Modern LLMs share a common transformer-family skeleton, differing mainly in trained weights, configuration settings, and post-training modifications.
The transformer architecture has become widely adopted across various AI domains, with emerging alternatives like state-space models potentially shaping future architectures.
