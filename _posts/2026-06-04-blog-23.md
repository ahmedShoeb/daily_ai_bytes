## Key Points

This is a comprehensive technical tutorial demonstrating how to use OpenAI models on Amazon Bedrock with the Responses API
The guide covers setting up Bedrock-hosted OpenAI models with API compatibility for production workflows
Key features include text generation, structured outputs, application tools, file inputs, response state management, and prompt caching
The tutorial walks through building a support assistant for BrightCart (fictional retailer) handling delayed and damaged-order replacement requests
Users learn configuration, verification, text requests, structured JSON generation, function calling, parallel tools, and custom tools
The guide explains direct file input, stateful/stateless conversation management, encrypted reasoning context, and background processing
Prompt caching techniques are demonstrated for latency optimization with different retention periods based on model versions
Operational smoke checks and cleanup procedures are included for production readiness
The tutorial emphasizes separating model-selected arguments from application-executed business logic using function calling
Real-world examples include PDF file processing, conversation state management, and support workflow implementations
