## Key Points

AWS has launched a new console experience in Amazon Bedrock optimized for the bedrock-mantle endpoint that supports GPT, Claude, and open-weight models with OpenAI and Anthropic APIs.
The new console features improved model discovery with a card-based catalog showing capabilities, modality support, context windows, and service quotas in a single view.
It introduces project-based workflows that allow users to organize evaluations and usage insights in a streamlined process mirroring generative AI application lifecycles.
The console provides live documentation with automatically prefilled code samples, SDK snippets, and API references that update based on project variables and settings.
Users can compare up to 3 models side-by-side to evaluate responses to the same prompt, aiding in model selection and optimization decisions.
The dashboard shows detailed usage analytics including token consumption patterns, inference request rates, and performance metrics to inform workload decisions.
Users can connect AI coding assistants like Claude Code, Cursor, and OpenCode directly to Bedrock with configuration guidance for routing requests through the bedrock-mantle engine.
The console supports both Anthropic API Protocol (for Claude features like Messages API) and OpenAI API Protocol (for features like Responses API) with automatically prefilled references.
The legacy Bedrock console remains available for managing features like Agents, Knowledge Bases, Guardrails, fine-tuning, and bedrock-runtime endpoint operations.
The new console is available in multiple AWS Regions worldwide including US East, US West, Asia Pacific, Europe, and South America locations where bedrock-mantle endpoints are offered.
