## Key Points

- FlashMemory-DeepSeek-V4 is a lightweight retriever that sparsifies DeepSeek-V4's Compressed-Sparse-Attention (CSA) KV-cache
- The retriever predicts which CSA KV-cache chunks the next ~64 tokens will attend to, allowing only top-scoring chunks to stay on GPU while others can be offloaded
- It matches or beats full-attention baselines while keeping only ~10-15% of the KV cache on-device, saving 80-90% of memory
- Uses a unique scoring system: hidden states are processed through LoRA layers, RMSNorm, and RoPE to compute per-chunk sigmoid scores
- The checkpoint contains three independent CSA layers (l10, l12, l20) whose scores are ensembled to make final keep/drop decisions
- Demonstrates impressive performance on reasoning-heavy long-context tasks up to 512K tokens with minimal accuracy loss
- Includes a toy implementation showing the decode control flow but is not a production KV swap engine or runnable DeepSeek-V4 model
- Specialized for ultra-long context applications where traditional attention mechanisms become computationally expensive
- Requires additional threshold-fallback mechanisms for precise needle-retrieval tasks like MRCR
